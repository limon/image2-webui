# image2-webui 功能现状说明

本文根据当前代码实现整理，目标是说明这个仓库现在已经具备的功能，而不是规划中的功能。

## 1. 项目概览

- 技术栈：`FastAPI + SQLite + 原生 HTML/CSS/JS`
- 入口页面：
  - `/`：主工作台
  - `/favorites`：收藏页
- 后端主文件：`app/main.py`
- 前端主文件：
  - `app/static/index.html`
  - `app/static/app.js`
  - `app/static/favorites.html`
  - `app/static/favorites.js`

这个项目本质上是一个“带任务记录和收藏能力的图像生成/编辑 Web UI”。前端负责参数输入、上传和结果展示，后端负责排队、调用上游图像接口、落盘、记录任务和收藏快照。

## 2. 主页面现有功能

### 2.1 三种任务模式

主页面目前有 3 个标签页：

1. 文生图 `generate`
2. 图像编辑 `edit`
3. 参考图生图 `reference`

三种模式共用一套任务历史、收藏、设置和调试能力。

### 2.2 文生图

支持能力：

- 输入 prompt 发起生成
- 选择尺寸：
  - 预设 1K / 2K / 4K
  - `auto`
  - 自定义尺寸
- 选择生成张数 `1-4`
- 支持结束正在运行的任务
- 支持重置当前表单
- 结果区展示生成中的预览图和最终图
- 支持结果大图预览、下载、收藏

尺寸校验规则：

- `auto` 直接放行
- 自定义尺寸必须是 `宽x高`
- 宽高都必须能被 `16` 整除
- 最长边不能超过 `3840`
- 总像素必须在 `0.7 MP - 8.85 MP` 之间

### 2.3 图像编辑

支持能力：

- 上传原图，支持点击、拖拽、`Ctrl+V` 粘贴
- 可选上传蒙版图
- 提供在线涂抹蒙版编辑器：
  - 涂抹
  - 擦除
  - 调整笔刷大小
  - 清空
  - 完成后回填到编辑表单
- 可选择“上传前缩小相关图片”
  - 原图和蒙版会按比例缩小到最长边不超过 `1536px`
  - 目的主要是节约 token / 上传体积
- 输入修改描述
- 选择尺寸、生成张数 `1-4`
- 支持结束任务、重置表单
- 结果区展示预览图和最终图

### 2.4 参考图生图

支持能力：

- 上传 1 张或多张参考图
- 支持点击、拖拽、`Ctrl+V` 粘贴
- 最多支持 `10` 张参考图
- 前端显示参考图缩略图
- 可选择“上传前缩小相关图片”
  - 参考图会按比例缩小到最长边不超过 `1536px`
- 输入生成 prompt
- 选择尺寸、生成张数 `1-4`
- 支持结束任务、重置表单
- 结果区展示预览图和最终图

## 3. 结果展示与任务交互

### 3.1 最近结果展示

首页顶部有最近任务展示区，会混合展示最近产生的任务结果。当前实现支持：

- 展示任务类型、状态、尺寸、张数等信息
- 展示失败提示和分槽位错误
- 查看详情
- 下载结果
- 收藏结果
- 复用任务配置

### 3.2 大图预览

结果图片支持 lightbox 预览，当前能力包括：

- 上一张 / 下一张切换
- 鼠标滚轮切换
- 键盘切换
- 在 lightbox 中直接收藏 / 取消收藏

### 3.3 任务状态恢复

前端会把当前活跃任务记录到浏览器本地存储。当前行为：

- 页面刷新后会尝试恢复任务追踪
- 继续轮询 `/api/jobs/{job_id}` 更新状态
- 如果任务已经结束，会同步更新界面并清理本地状态

## 4. 历史记录与回收站

主页面提供历史记录弹层。

### 4.1 历史记录

支持能力：

- 查看任务历史
- 按 prompt 关键词搜索
- 按条件筛选：
  - 全部
  - 完成
  - 失败
  - 文生图
  - 编辑
  - 参考图
- 打开任务详情
- 下载结果
- 复用配置
- 删除失败任务
- 清空历史

### 4.2 回收站

任务删除并不是立即物理删除，而是先进入回收站。

支持能力：

- 查看回收站任务
- 从回收站恢复任务
- 清空回收站

当前实现里：

- `DELETE /api/jobs/{job_id}` 是“移入回收站”
- `DELETE /api/jobs?trashed=1` 才是永久清空回收站
- 永久清理时会删除对应任务文件目录

## 5. 收藏系统

### 5.1 收藏入口

当前可以从以下位置收藏结果图：

- 首页结果卡片
- lightbox 预览

### 5.2 收藏的实现方式

收藏不是只记一个原图链接，而是创建一份“快照”：

- 复制最终图到 `app/data/files/favorites/...`
- 尽量复制关联素材：
  - 编辑任务的原图 / 蒙版
  - 参考图生图的参考图
- 在 SQLite 中记录收藏元数据

因此即使原任务后续被清理，收藏本身仍然可以继续存在。

### 5.3 收藏页

`/favorites` 页面当前支持：

- 查看所有未归档收藏
- 查看已归档收藏
- 两种布局：
  - 瀑布流 `masonry`
  - 网格 `grid`
- 查看收藏详情
- 查看关联原图 / 蒙版 / 参考图
- 下载收藏图
- 复用收藏对应的任务配置
- 取消收藏
- 恢复已归档收藏

当前实现里：

- 删除收藏实际是“归档”
- 归档收藏可以恢复
- 收藏页会自动尝试迁移旧版浏览器本地收藏数据到服务端

## 6. 设置与本地持久化

设置面板当前支持：

- 主题切换：
  - 跟随系统
  - 浅色
  - 深色
- 保存 API Key
- 保存 API Base URL
- 保存 Model
- 选择 `quality`
  - `high / medium / low / auto`
- 选择 `moderation`
  - `low / auto`
- 选择多图模式：
  - `fanout`：拆成多个 `n=1` 并行请求
  - `direct`：单请求直接传 `n`

本地存储内容主要包括：

- API Key
- Base URL
- Model
- Quality
- Moderation
- 主题
- 当前活跃任务
- 收藏页布局
- 待复用任务快照

注意：

- API Key 是保存在浏览器本地，不是服务端数据库
- 后端只在任务执行期间把 API Key 暂存到内存

## 7. 调试能力

首页有全局 DEBUG 面板。

当前支持：

- 聚合显示多个任务的调试日志
- 显示时间、任务 ID、日志级别
- 清空当前可见日志
- 折叠 / 展开调试面板

后端会记录的调试信息包括：

- 任务提交和排队
- 服务端开始处理
- 当前使用的上游调用方式
- 多图并发执行情况
- 部分成功修复信息
- 成功、失败、取消等状态

## 8. 后端任务处理现状

### 8.1 任务存储

当前所有任务和收藏元数据都放在：

- `app/data/jobs/jobs.sqlite3`

任务记录包含：

- 基本参数：类型、prompt、尺寸、模型、数量
- 上游设置：`base_url`、`quality`、`moderation`、`batch_mode`
- 状态字段：`queued / running / succeeded / failed / cancelled`
- 结果字段：预览图、最终图、多张结果数组
- 调试日志
- 槽位级错误、改写词、预览阶段
- 回收站状态
- 时间信息和耗时

### 8.2 文件存储

当前文件落盘位置：

- 任务文件：`app/data/files/<job_id>/...`
- 收藏快照：`app/data/files/favorites/<favorite_id>/...`

主要包含：

- 上传的原图 / 参考图 / 蒙版
- 中间预览图
- 最终结果图
- 收藏快照复制品

### 8.3 并发与恢复逻辑

当前后端行为：

- 同时最多处理 `2` 个任务
- 服务启动时会把未完成的旧任务标记为失败
- 服务启动时会尝试修复“历史上已经产出部分结果但状态仍是 failed”的任务

### 8.4 上游调用策略

当前模型判断逻辑：

- 如果 `model` 以 `gpt-image-` 开头，走原生图像接口
  - 文生图：`/images/generations`
  - 编辑：`/images/edits`
- 否则走 `/responses`，由后端构造 `image_generation` 请求

多图任务策略：

- `fanout`：拆成多个子请求并行执行
- `direct`：单个请求直接传 `n`

### 8.5 可取消任务

当前仅 `queued` 和 `running` 状态允许取消。

取消后：

- 状态改为 `cancelled`
- 未成功产出的槽位会补上取消说明
- 前端停止追踪

## 9. 当前接口清单

### 9.1 页面接口

- `GET /`：主页面
- `GET /favorites`：收藏页面

### 9.2 任务接口

- `GET /api/jobs/count`：获取历史和回收站数量
- `GET /api/jobs`：获取任务列表
- `GET /api/jobs/{job_id}`：获取单个任务
- `POST /api/jobs/generate`：提交文生图任务
- `POST /api/jobs/edit`：提交图像编辑任务
- `POST /api/jobs/reference`：提交参考图生图任务
- `POST /api/jobs/{job_id}/cancel`：取消任务
- `DELETE /api/jobs/{job_id}`：移入回收站
- `POST /api/jobs/{job_id}/restore`：从回收站恢复
- `DELETE /api/jobs`：清空任务，或按 `status=failed` 清理失败任务

### 9.3 收藏接口

- `GET /api/favorites`：获取收藏列表
- `GET /api/favorites/{favorite_id}`：获取单个收藏详情
- `POST /api/favorites`：从任务结果创建收藏快照
- `POST /api/favorites/import`：导入旧收藏数据
- `DELETE /api/favorites/{favorite_id}`：归档收藏
- `POST /api/favorites/{favorite_id}/restore`：恢复收藏
- `DELETE /api/favorites`：归档全部收藏

## 10. 运行方式

### 10.1 本地开发

在 `app` 目录下：

```bash
pnpm install
pnpm dev
```

或直接启动：

```bash
pnpm start
```

默认端口：`8000`

### 10.2 Docker

项目根目录提供：

- `Dockerfile`
- `docker-compose.yml`

容器会把 `./app/data` 挂载到容器内 `/app/data`，用于持久化任务和收藏数据。

## 11. 当前限制与注意事项

- Python 要求 `>= 3.13`
- 参考图最多 `10` 张
- 单次生成张数最多 `4` 张
- 自定义尺寸受 16 对齐、最长边和总像素限制
- 活跃任务只在浏览器本地做恢复追踪，不是服务端主动推送
- 收藏删除是归档，不是立即硬删除
- 任务删除先进入回收站，永久清理才会删文件
- API Key 不落库，但会在当前服务进程内存中临时持有到任务结束
